# Auxiliary tools for protomidpy

### Weight correction
Code for creating `.npz` files from ALMA Measurement Sets (MS), and for estimating and correcting biases in `WEIGHT` using disk geometry.

* `ms_to_npz.py` — Converts a Measurement Set into an `.npz` file. It can also be used to generate input files for `protomidpy`.
* `compute_bias_weight.py` — Estimates the bias in the weights under the assumption of axisymmetry, using disk geometry (center, cosi, and PA).
* `correct_weight_ms.py` — Corrects the `WEIGHT` column in the MS based on the estimated bias (and optionally `SIGMA` as well).
* `average_ms.py` — Performs channel and time averaging on a Measurement Set. Useful for reducing the data volume.

### Producing a residual Measurement Set
* `ms_to_npz_spw.py` — Outputs the `(u,v)` coordinates of a Measurement Set as arrays for each SPW. This is required for residual calculations.
* `model_calc_spw.py` — Computes the model from `protomidpy` results. It also generates the model in a format matched to the Measurement Set.
* `make_residual_ms.py` — Subtracts the visibility model generated by `protomidpy` from the Measurement Set data.

## Quick Start
---

- Weight correction

```bash
# 1) Convert MS to NPZ
casa -c ms_to_npz.py  # Run this directly from the terminal without launching CASA interactively

The resulting `.npz` file can also be used as input for `protomidpy`.

# 2) Estimate the WEIGHT bias (disk geometry is required)
python compute_bias_weight.py

# 3) Apply the bias correction
casa -c correct_weight_ms.py  # Run this directly from the terminal without launching CASA interactively
```

## Tips
- Replace the MS filename, output filename, and disk geometry with your own values.
- You can use `average_ms.py` to average the MS and reduce its size, which makes the analysis easier.

- Producing Residual Measurement set
```bash
# 1) Convert MS to NPZ
casa -c ms_to_npz_for_spw.py

# 2) Compute the model with protomidpy
#    (required in order to subtract the model from the MS)
python model_calc_spw.py

# 3) Subtract the model from the MS
casa -c make_residual_ms.py  # Run this directly from the terminal without launching CASA interactively

- modelfile:
  Path to the model file from `protomidpy` (the `model.npz` file created by `model_calc.py`)

- msfile:
  Path to the original Measurement Set analyzed with `protomidpy`

- cos_dec:
  cos(declination), where declination is the Dec of the target (RA, Dec) in radians
```
